# Delayed Proxies

Delayed proxies are a Bittensor wallet-security pattern for proxied actions that should not execute immediately. The official [Delayed Proxies](https://docs.learnbittensor.org/keys/proxies/delayed-proxies) and [Proxy Types](https://docs.learnbittensor.org/keys/proxies/proxy-types) docs describe them as a way to add a review window before a sensitive proxied call can execute.

## What the Delay Protects

The main benefit is the review window. A delayed proxy does not remove authority, but it gives operators time to review a queued action and cancel it before execution if the action is wrong, unsafe, or no longer intended.

## Distinction from Pure Proxies

[[Pure Proxies]] are about stable keyless account structure, especially for shared custody patterns. Delayed proxies are about time-buffered execution and review, not identity continuity.

## Distinction from Multisig Wallets

[[Multisig Wallets]] require threshold approval from several signers. A delayed proxy can still be useful when one operator path exists but the action should wait through a review window before execution.

## Reader Boundary

Readers should connect delayed proxies to [[Coldkey and Hotkey Workstation Security]] rather than treat them as a generic wallet convenience feature. The core idea is operational safety: create a review window, inspect the queued action, and cancel it during that delay period when needed.
